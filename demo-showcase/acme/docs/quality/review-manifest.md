## Reviewer checklist

Use this checklist before approving the documentation for publish. Every item must be checked by the reviewing operator.

- [ ] Confirm stage summary has no missing **required** artifacts (consolidated report and audit scorecard are optional for demo).
- [ ] Review all five protocol contract reports and verify 4/5 pass (gRPC failure is a known demo limitation).
- [ ] Review the KPI wall and confirm quality score trend is stable or improving.
- [ ] Check the gap analysis and confirm no new high-priority gaps appeared since last review.
- [ ] Review the retrieval evaluation report and confirm precision/recall are acceptable for the use case.
- [ ] Verify all documentation pages have valid frontmatter (100% metadata completeness).
- [ ] Confirm the MkDocs site builds without errors.
- [ ] Approve publish only if critical findings are resolved or tracked as known issues.
