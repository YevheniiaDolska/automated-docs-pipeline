# Intent experiences reference

Intent experiences are generated pages and channel bundles that combine active knowledge modules for one intent and audience.

```bash
ls docs/reference/intent-experiences/
```

## Generated docs pages

- `configure-operator.md`: generated by intent assembler for documentation channel.

## Generated bundle outputs

- `reports/intent-bundles/*-assistant.json`
- `reports/intent-bundles/*-automation.json`
- `reports/intent-bundles/*-in-product.json`

## Retrieval artifact

- `docs/assets/knowledge-retrieval-index.json`: module-level retrieval records for assistants and search.
- `docs/assets/knowledge-graph.jsonld`: lightweight JSON-LD graph for module relationships.
- `reports/retrieval_evals_report.json`: Precision/Recall/Hallucination-rate metrics.

## Source of truth

- `knowledge_modules/*.yml`: module definitions with metadata and channel-specific content.
- `knowledge-module-schema.yml`: schema contract for module structure.

## Validation commands

```bash
npm run lint:knowledge
npm run validate:knowledge
```
